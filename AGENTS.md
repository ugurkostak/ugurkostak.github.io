# AGENTS.md — ugurkostak.github.io

## Project Overview

This repository is a static GitHub Pages personal website for Ugur Kostak. It presents photography, cinema writing, technology writing, algorithmic-art explorations, and soft-skills/professional-growth writing.

The site should remain lightweight, maintainable, accessible, and easy to update without a build system.

## Tech Stack

- Static HTML pages in the repository root and section folders
- Vanilla JavaScript
- Bootstrap 3 style/layout conventions
- Font Awesome icons
- CSS, JavaScript, fonts, and images stored under `assets/`
- Python utility scripts under `scripts/`
- Metadata JSON files for blog/gallery entries
- GitHub Pages deployment

## Current Architecture Principles

- Main HTML pages live in the repository root.
- Section subpages live inside active content folders such as `cinema/`, `tech-blog/`, `algorithmic-art/`, and `photography/`.
- Reusable navigation, sidebar header, mobile bar, and footer are generated by `assets/js/sidebar.js` or the project’s current sidebar script.
- Individual HTML pages should not duplicate the full sidebar/header/footer markup.
- Every HTML page should use:

```html
<header id="site-header"></header>
```

- Page-specific content belongs inside:

```html
<main id="main-collapse">
  <!-- page content -->
</main>
```

## Agent Rules

### General

- Understand nearby files before editing.
- Prefer small, reviewable changes.
- Do not introduce frameworks, package managers, bundlers, or dependencies unless explicitly requested.
- Preserve the visual identity unless the task explicitly asks for redesign.
- Keep paths relative and compatible with GitHub Pages.
- Do not remove existing content unless requested.
- Treat soft-skills writing as reflective, practical content about communication, collaboration, feedback, mentoring, learning, teamwork, leadership, and career growth.

### HTML

- Use semantic HTML.
- Preserve the page skeleton and reusable layout placeholder.
- Do not paste full sidebar/header/footer HTML into every page.
- Keep heading hierarchy logical.
- Add accessible alt text to meaningful images.

### JavaScript

- Use vanilla JavaScript.
- Keep reusable layout/navigation logic in `assets/js/sidebar.js` or the existing sidebar file until the move is completed.
- Keep functions small and readable.
- Avoid global state unless the existing template requires it.
- Preserve compatibility with the existing template functions such as sidebar toggle behavior.

### CSS and Assets

- Keep CSS files under `assets/css/`.
- Keep JavaScript files under `assets/js/`.
- Keep fonts under `assets/fonts/`.
- Keep images under `assets/images/<section>/`.
- Use unified blog styling: `assets/css/blog.css` and `assets/js/blog.js` for all collection pages (tech-blog, cinema, photography, etc.).
- Prefer adding clear custom CSS instead of editing minified vendor/template files when readable source exists.

### Metadata

- Blog/gallery sections should use `metadata.json` for listing, sorting, filtering, and presentation.
- Metadata should be human-readable and stable.
- Do not rename slugs casually because URLs may break.
- Each metadata entry should point to an existing HTML page and image path.
- **Articles must be ordered by date (newest first).**

### Images

- Store original/high-resolution photos in `assets/images/photos/big/`.
- Store generated web-sized photos in `assets/images/photos/small/`.
- Use `scripts/shape_images.py` to proportionally downscale images.
- Do not overwrite originals.

## Validation Checklist

Before finishing a change, check:

- Page loads without JavaScript errors.
- Desktop sidebar works.
- Mobile navbar toggle works.
- Active navigation state is correct.
- Relative paths work from root pages and nested section pages.
- Images load from `assets/images/...`.
- Metadata JSON remains valid.
- No obvious accessibility, SEO, or mobile responsiveness regression.
